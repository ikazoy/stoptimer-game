# ストップタイマーゲーム 要件・設計・実装タスク

## 目的

5歳から12歳の子供が、iPad横向きで遊ぶストップタイマーゲームを作る。スタッフは事前に難易度プリセットを作り、プレイヤーはそのプリセットを選んで「指定秒数ぴったり」を狙う。

## 配布とオフライン方針

- Webアプリとして配布し、iPadではホーム画面に追加したPWAとして利用する。
- 初回だけはオンラインでアクセスし、ホーム画面追加と一度の起動確認を済ませる運用にする。
- ゲーム中はネットワーク不要。アプリ本体はService Workerでキャッシュし、設定データはローカル保存する。
- 会場前に「オンライン起動」「PWA起動」「オフライン起動」を確認する。
- 更新はオンライン時の再読み込みで反映する。ゲーム中に自動で画面を差し替えない。

## 画面構成

- プレイヤー用トップ: プリセットを大きなカードボタンで表示し、右上の歯車タップで管理者画面に入る。
- 管理者PIN画面: 初期PINは `0000`。本格的な認証ではなく、子供の誤操作防止を目的とする。
- ゲーム画面: 「スタート」後に計測を開始し、中央の大きな「ストップ！」ボタンで停止する。
- 結果画面: 実測秒数、目標との差、賞、演出、「もう一回」「メニューへ」を表示する。
- 管理者メニュー: プリセットの追加、編集、削除、リネーム、JSON import/export、管理者PIN変更を行う。

## プリセット仕様

```json
{
  "id": "preset-id",
  "name": "ふつう 10秒",
  "targetSeconds": 10,
  "visibleUntilSeconds": 7,
  "firstPrizeError": 0.2,
  "secondPrizeError": 0.5,
  "thirdPrizeError": 1,
  "themeColor": "#2f80ed"
}
```

### 判定仕様

- 誤差は `abs(実測秒数 - 目標秒数)` で判定する。
- 賞は `1等 / 2等 / 3等` の3種類。
- 残念賞は出さない。全員が必ず3等以上になる。
- `1等 <= 2等 <= 3等` のしきい値順を必須にする。
- 3等しきい値を超えた場合も3等として扱う。
- 目標秒数はプリセットごとに変更できる。5秒、7秒、10秒など任意の正の秒数を設定できる。
- 表示秒数は目標秒数以下にする。目標秒数と同じ値にすると、最後までカウントを表示する。

### デフォルトプリセット

| 名前 | 目標 | 表示 | 1等 | 2等 | 3等 |
| --- | ---: | ---: | ---: | ---: | ---: |
| かんたん 10秒 | 10.0秒 | 10.0秒まで | ±0.3秒 | ±0.7秒 | ±1.2秒 |
| ふつう 10秒 | 10.0秒 | 7.0秒まで | ±0.2秒 | ±0.5秒 | ±1.0秒 |
| むずかしい 15秒 | 15.0秒 | 3.0秒まで | ±0.2秒 | ±0.6秒 | ±1.5秒 |

## ローカル保存

- プリセットと管理者PINは `localStorage` に保存する。
- データ量が小さく、ランキングや履歴保存が不要なため、初期実装ではIndexedDBを使わない。
- JSON import/exportにより、別iPadへの設定移行やバックアップを行える。

## JSON import/export

- 管理者向けの裏機能として提供する。
- JSONテキスト貼り付けと `.json` ファイル選択に対応する。
- import時は「追加」と「置き換え」を選べる。
- export時は現在のプリセットをJSONとして保存できる。

## UI方針

- iPad横向きを主対象にする。
- 縦向きでは横向き案内を表示する。
- 子供が迷わないよう、大きなボタン、大きな文字、はっきりした色を使う。
- 結果画面は賞に応じてリッチな演出を出す。
- 音はゲーム操作後の効果音として鳴らす。アプリ内の音ON/OFF設定は持たず、iPad本体側で制御する。
- 色だけで結果を伝えず、賞名と数値を大きく表示する。

## 技術スタック

- Vite
- React
- TypeScript
- plain CSS
- vite-plugin-pwa
- Playwright

サーバー側実装は持たない。静的ホスティングに配置できる構成にする。

## E2Eテスト範囲

- デフォルトプリセットが表示される。
- プリセット選択、スタート、ストップ、結果表示、もう一回が動く。
- 管理者PINで管理画面に入れる。
- プリセットの追加、編集、削除ができる。
- JSON importが追加・置き換えで動く。
- JSON exportが動く。
- PWAとしてService Workerが登録され、オフライン再読み込み相当が動く。

## 実装タスク

1. Vite + React + TypeScriptの初期構成を作る。
2. 要件と設計をこの文書に固定する。
3. プリセット型、デフォルト値、バリデーション、localStorage保存を実装する。
4. プレイヤー用トップを実装する。
5. 管理者PIN導線を実装する。
6. プリセット管理画面を実装する。
7. JSON import/exportを実装する。
8. ゲーム計測画面を実装する。
9. 結果画面と演出を実装する。
10. PWA manifestとService Workerを設定する。
11. Playwright E2Eを実装する。
12. build、lint、E2Eを実行して確認する。
13. ローカルサーバーを起動し、動作確認用URLを提示する。
